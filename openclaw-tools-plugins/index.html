Tool e plugin

Come collegare tool e plugin in OpenClaw senza trasformare il setup in caos

In OpenClaw i plugin non sono un dettaglio estetico: possono aggiungere tool, comandi CLI, route HTTP, servizi e altre estensioni. Per questo il punto non e installarne tanti, ma capire quando servono davvero e come non rompere il gateway.

La differenza pratica

I tool core fanno gia parte di OpenClaw. I plugin invece estendono il sistema e possono introdurre comandi, agent tools, integrazioni e servizi in background. Nei docs ufficiali il comando base da cui partire e questo:

openclaw plugins list

Il percorso piu pulito

  1. Guarda cosa e gia caricato con openclaw plugins list.
  2. Installa un plugin ufficiale solo se serve davvero al primo workflow.
  3. Riavvia il gateway e configura il plugin nella sua entry dedicata.
openclaw plugins install @openclaw/voice-call
openclaw plugins enable voice-call
openclaw plugins doctor

Un dettaglio importante di sicurezza

La documentazione OpenClaw e molto chiara: i plugin nativi girano in-process con il gateway e vanno trattati come codice trusted. Questo vuol dire che un plugin mal fatto puo rompere o destabilizzare il sistema. Quindi:

  • installa solo plugin che capisci o di cui ti fidi
  • non aggiungere plugin prima di avere un primo workflow chiaro
  • usa plugins.allow o una selezione minima quando il setup diventa piu serio

Quando i plugin servono davvero

Servono quando il caso d'uso richiede una capacita non inclusa nel core, per esempio una integrazione specifica o un canale opzionale. Non servono per "fare piu AI". Il criterio giusto e sempre questo: prima fai funzionare OpenClaw, poi modelli, poi un agente, poi eventualmente i plugin strettamente utili.

Ordine consigliato

  • Installa OpenClaw
  • Collega un modello
  • Valida il primo agente
  • Controlla il troubleshooting
