Troubleshooting

هل OpenClaw لا يعمل؟ ابدأ من هذه الفحوصات قبل أي إصلاح عشوائي

وثائق OpenClaw الرسمية تتضمن سلّم triage مفيد جدًا. إذا اتبعته بالترتيب ستعرف بسرعة هل المشكلة في الـ gateway أو القنوات أو النموذج أو الـ dashboard أو في النظام نفسه. هذه هي المختصر العملي.

قائمة أول 60 ثانية

openclaw status
openclaw status --all
openclaw gateway probe
openclaw gateway status
openclaw doctor
openclaw channels status --probe
openclaw logs --follow

هذا التسلسل موصى به في الوثائق الرسمية لعمل triage سريع. إذا فشل شيء هنا، فلا معنى لتغيير الإضافات أو النماذج عشوائيًا.

إذا لم يتصل الـ dashboard أو Control UI

  • يجب أن يظهر openclaw gateway status أن الـ runtime نشط وأن probe الـ RPC سليم.
  • إذا ظهرت أخطاء auth فالمشكلة قد تكون في الـ token أو وضعية المصادقة.
  • إذا ظهرت أخطاء connect فغالبًا السبب URL أو port خاطئ.

أعد ترتيب الـ gateway والواجهة أولاً قبل تغيير أي شيء آخر.

إذا كان الـ gateway لا يبدأ

  • port مستخدم مسبقًا
  • gateway mode غير صحيح
  • bind بلا auth على واجهات غير محلية

الأوامر الأساسية تبقى نفسها: openclaw gateway status و openclaw doctor ومتابعة الـ logs.

إذا بدا أن المشكلة في النموذج أو الإضافة

openclaw models status
openclaw models list
openclaw plugins list
openclaw plugins doctor
  • إذا كان النموذج محجوبًا فقد تكون المشكلة في allowlist داخل agents.defaults.models.
  • إذا كانت الإضافة مثبتة لكنها لا تُحمّل فقد ينقصها manifest صالح أو config صحيح.
  • تذكّر أن الإضافات الأصلية تعمل داخل الـ gateway نفسه، لذلك قد تؤدي إضافة سيئة إلى زعزعة كامل النظام.
