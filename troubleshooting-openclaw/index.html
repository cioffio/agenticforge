Troubleshooting

OpenClaw non funziona? Parti da questi controlli invece di tentare fix a caso

La documentazione ufficiale OpenClaw ha una sequenza di triage molto utile: se la segui in ordine, capisci rapidamente se il problema e nel gateway, nei canali, nel modello, nella dashboard o nel sistema. Questa e la scorciatoia pratica.

La checklist dei primi 60 secondi

openclaw status
openclaw status --all
openclaw gateway probe
openclaw gateway status
openclaw doctor
openclaw channels status --probe
openclaw logs --follow

Questa scala viene consigliata direttamente nei docs OpenClaw per fare triage rapido. Se qui qualcosa fallisce, non ha senso continuare a cambiare plugin o modelli alla cieca.

Se dashboard o Control UI non si collegano

  • openclaw gateway status dovrebbe mostrare runtime attivo e probe RPC ok.
  • Se vedi errori di auth, il problema puo essere nel token o nella modalita di autenticazione.
  • Se vedi errori di connect, spesso il problema e un URL o una porta sbagliata.

Prima di toccare altro, rimetti in ordine gateway e UI.

Se il gateway non parte o il servizio non gira

Qui i segnali tipici sono:

  • porta gia occupata
  • gateway mode non corretto
  • bind senza auth su interfacce non locali

I comandi chiave restano openclaw gateway status, openclaw doctor e i log in follow.

Se il problema sembra nei modelli o nei plugin

openclaw models status
openclaw models list
openclaw plugins list
openclaw plugins doctor
  • Se un modello non e consentito, il nodo puo essere nell'allowlist di agents.defaults.models.
  • Se un plugin e installato ma non carica, puo mancare manifest valido o config corretta.
  • Ricorda che i plugin in OpenClaw girano in-process: se il plugin e sbagliato puo destabilizzare il gateway.
